# CLAUDE.md — task-board

## プロジェクト概要

タスク管理ボードアプリケーション。

## Git 運用ルール

### 基本方針
- コードを変更するたびに GitHub へプッシュする
- `main` ブランチへの直接プッシュを基本とする（規模が大きい機能追加はブランチを切る）

### コミット手順

1. 変更をステージング（関係するファイルを個別に指定する）
2. 変更内容を表すコミットメッセージを書く
3. プッシュする

```bash
git add <変更したファイル>
git commit -m "変更内容の説明"
git push origin main
```

### コミットメッセージの書き方
- 日本語・英語どちらも可
- 「何を」ではなく「なぜ」を軸に書く
- 例: `fix: ボタンクリック後にページがリロードされる問題を修正`

### 禁止事項
- `--no-verify` によるフック無効化
- `--force` プッシュ（明示的な指示がない限り）
- `.env` や認証情報ファイルのコミット

## コーディング規約

- コメントは WHY が自明でない場合のみ書く（WHAT は書かない）
- エラーハンドリングはシステム境界（ユーザー入力・外部 API）にのみ追加する
- 不要な抽象化・将来の要件への備えはしない

## 開発フロー

1. 変更を加える
2. 動作確認する
3. `git add` → `git commit` → `git push` の順で GitHub へ反映する
